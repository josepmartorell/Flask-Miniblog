NEWS of the Spiderweb

Web Scrapers 2.0 This project consists of developing a specialized automatic webScraping diary (as would be the example of this website) for this platform. Said digital journal would be updated by means of a Python application created with the Scrapy framework which would be hosted on Heroku and using the patch called Scrapyd. Through this patch it is intended to use, while the budget does not allow it, the same services of the ScrapingHub platform to automatically fire the spiders according to a programming schedule only at zero cost, even when the quality is... not the same. Simply to test this service and improve its performance.
